File System Access

chrome.fileSystem

  • Demo
  • Code

chrome.fileSystem.chooseEntry({ 
  type: 'openFile', 
  accepts: [{ extensions: [ 'jpg', 'jpeg', 'gif', 'png' ] }]}, 
  function(entry) {
    // this gives us a file entry. We just need to read it.
    entry.file(function(file) {
      // create a new file reader
      var reader = new FileReader();
      // create an event for when the file is done reading
      reader.onloadend = function() {
        // set the source of the image to the result (base64)
        img = document.getElementById("demo-img");
        img.src = this.result;
      };
      // read the file as a data URL
      reader.readAsDataURL(file);
  });
});

Idle Status

chrome.idle

  • Demo
  • Code

User Idle Status:

The user was detected as "idle" after 15 seconds of inactivity.

var idleState = document.getElementById("#idle-state");

// the window can be set for as low as 15 seconds
chrome.idle.setDetectionInterval(15);

// when the window is broken, the state changes
chrome.idle.onStateChanged.addListener(function(state) {
  idleState.append(document.createTextNode(state));
});

it probably has a Content Security Policy.

Remember This?

"Packaged apps deliver an experience as capable as a native app, but as safe as a web page."

There Will Be ABSOLUTELY No...

  • Inline execution of script
  • Use of eval
  • Construction of new Function
  • Displaying external content through iframe's
    • Although the webview is provided as an alternative
  • More stuff that I can't remember...

This Sucks

The Sandbox

  • A page in your app that is not restricted by CSP
  • There may be more than 1
  • They must be declared as sandbox pages in the manifest
  • They have no access to any native API's
